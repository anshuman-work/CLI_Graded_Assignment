# Question 4 - Real-Time Log Monitoring Using Linux Command Pipeline

## Objective

Design a Linux command pipeline that:

- Displays newly added log entries in real time.
- Extracts only ERROR messages.
- Stores ERROR messages in a separate report file.
- Suppresses unnecessary error output.
- Demonstrates the use of pipes, grep, tail, redirection, and `/dev/null`.

---

### 4. Start real-time monitoring

```bash
tail -n 0 -f system.log | grep --line-buffered "ERROR" > error_report.txt 2>/dev/null
```

**Explanation:**
Started monitoring the log file in real time. Only newly added lines containing the word `ERROR` were written to `error_report.txt`, while error messages generated by commands were redirected to `/dev/null`.

**Explanation:**
Displayed the extracted ERROR messages that were captured by the monitoring pipeline.

## Command Pipeline Explanation

### tail -n 0 -f

Continuously monitors the log file and displays only new lines added after monitoring begins.

### grep --line-buffered

Filters the incoming log entries and passes only the lines containing the word `ERROR`. The `--line-buffered` option ensures that matching lines are written immediately.

### Pipe (|)

Transfers the output from `tail` directly to `grep` without creating temporary files.

### Output Redirection (>)

Stores the filtered ERROR messages into `error_report.txt`.

### Error Redirection (2>/dev/null)

Suppresses unnecessary error messages by redirecting the standard error stream to `/dev/null`.

### /dev/null

A special Linux device that discards any data written to it, keeping the terminal output clean.

---

## How the Monitoring Tool Works

The monitoring command continuously watches the log file for newly added entries. Whenever a new line containing `ERROR` is appended, it is immediately filtered by `grep` and written to `error_report.txt`. Any unnecessary error output is discarded using `/dev/null`.
